# ADR-054: M8.1 UI & navigation consolidation (Explore, Captures, subtree feed, universal time/node affordances)

auto-recorded trust loop) ¬∑ **Supersedes** the T5-logged "true subtree grouping deferred" minor and the

## Context

Post-M8 live use surfaced five UI/navigation frictions: relative timestamps hide the exact time;
pipeline step runs flood the feed as flat interleaved rows (children sort above their parent under
`ts DESC`); Search and Map are two tabs where Map already embeds a (degraded) search; the Capture
tab's Recents is a flat 20 with no path to history while the Activity feed shows captures only as
organize-run telemetry; and node references are inert text on several surfaces. Grilled 2026-07-17.

## Decisions

**1. Exact time everywhere via one `<TimeAgo>` primitive.** The shared `relativeTime` helper is
wrapped in a `<TimeAgo iso>` component rendering the relative phrase plus a **custom tooltip ‚Äî tap
(mobile) and hover (desktop)** ‚Äî showing the absolute local time (`17 Jul 2026, 08:36`; 24h, no
seconds). Native `title` was rejected (invisible on touch). Every timestamp surface swaps to it.

**2. Pipeline subtrees leave the flat feed (server-side).** The feed's `agent_runs` branch filters
`parent_run_id IS NULL` ‚Äî a pipeline run is **one row** (its rollup summary already carries the
counts). `GET /activity/runs/{id}` gains a **recursive `children[]` tree** (each child: id, name,
status, ts, summary, its own `children[]`), so nested runs (e.g. distiller-spawned `capture` runs
under a step) reflect their true depth **in the data structure**. The client renders one expanded,
depth-indented tree, ordered **ascending (early‚Üílate, top‚Üíbottom)** at every level ‚Äî the natural
reading order of a pipeline. Client-side grouping of loaded rows was rejected: keyset pages can
split a parent from its children (the exact reason T5 deferred grouping). `parent_ref` stays on the
wire for compatibility but no longer drives rendering.

**3. Search + Map merge into one "Explore" tab (7‚Üí6 tabs).** Landing = search box; results are
today's full cards (type icon, plane badge, snippet, preview drawer); **picking a hit centers the
map constellation** (canvas/list toggle, breadcrumbs, restore-last-centered unchanged); a search
affordance stays reachable from anywhere in the explorer ‚Äî the results-list flow is first-class,
never only an empty state. **The plane/type filter chips die with the Search tab**: at personal
scale the semantic query does the narrowing; the taxonomy stays load-bearing product-wide
(organizer, map colors, retrieval, LLM digestion) and stays visible passively (icons/badges), but a
manual pre-filter UI earns nothing. The API keeps accepting `types`/`planes` ‚Äî UI removal, not a
contract change. **Chat's plane chips stay** (occasionally purposeful: "answer from this life area
only"; passive when unused).

**4. Captures become first-class activity content.** The feed's **Conversations category becomes
"Captures"**: all captures regardless of source (voice / text / MCP / chat), keyset-paginated,
each row expandable to full detail (raw text, resulting node chips, status, source badge);
chat-sourced rows keep the one-tap **Remove** (the M6 trust loop moves in with them ‚Äî a strict
superset, so a 4th tab was rejected). Organize **runs** stay in Agents & jobs: runs are machinery,
captures are content ‚Äî no duplication within a tab. The Capture tab's Recents shrinks to **~5**
with in-place expand + a "see all ‚Üí Activity" link.

**5. Universal clickable node references.** One shared **`NodeChip`** primitive (type icon +
title): every node reference anywhere ‚Äî capture node chips, review cards, graph-health offender
samples, Captures rows, feed details ‚Äî is clickable and **opens the shared `NodePreview` drawer**
(in-context card), which always offers **"Explore in map"** as the second hop. Uniform
click‚Üícard‚Üímap everywhere; straight-to-map was rejected (loses your place for what's usually a
glance). Consequence: **graph-health offender samples must carry node ids** in
`agent_runs.details` (today they are display strings) ‚Äî a small server-side payload change.

## Consequences

- Server changes are small but real (feed filter, recursive run detail, captures branch widening +
  detail fetch, graph-health ids) ‚Äî M8.1 is *not* web-only; 03-api is updated alongside.
- The feed gets dramatically quieter (12 step rows/night collapse into 1) while depth becomes
  *more* visible on demand, not less.
- Search's `features/search/` folder dissolves into `features/map/` (‚Üí Explore); the `AppShell`
  drops to 6 tabs.
- Rejected: native `title` tooltips (no touch), client-side subtree grouping (pagination-fragile),
  a 4th "Captures" tab (leaves Conversations a shell), per-context node-click behavior
  (inconsistent muscle memory).
