## Task 1

**Traverse primitive (`GraphService.neighbors`) + `build_context` core**
([ADR-046](../adr/046-m5-mcp-server-oauth-connectors.md) / [ADR-028](../adr/028-one-service-layer-mcp-peer-surface.md) /
[ADR-032](../adr/032-prior-art-adoptions.md)) — service layer only, **local, not pushed**. Commits
`ff4a729` (feature) + `bb33ae7` (review fixes).

The read side of the graph, thin over the `edges` table with **no LLM call**, shared by the MCP
`traverse`/`build_context` tools (task 4) and the M7 map endpoint — one primitive, no traversal
logic living in the tools.

**Store — `PgNeighborStore.neighbors`** (`server/app/graph/store.py`, plain asyncpg / no ORM,
rule 5). A center node's 1-hop neighbors, **both directions and both origins** unioned (the `out`
leg joins the `dst` endpoint, the `in` leg the `src`), filtered by optional `rel` + `direction`,
**keyset-paginated** on `(origin, rel, dir, node_id)` — the exact `ORDER BY`, so "next page" is
every row strictly greater than the last returned (`(origin,rel,dir,node_id) > (…::uuid)` row-value
comparison; the whole `after` tuple is NULL on page 1 via a `$::text IS NULL` guard). That key is
unique per center (edges PK `(src_id,dst_id,rel,origin)` + `dir`), so pages never skip or duplicate
and pagination is stable against concurrent edge writes. **Tombstoned endpoints excluded on both
ends** (`n.merged_into IS NULL` on each leg's join — ADR-030 §5); the center itself may be a
live-resolving tombstone (a `get_node` concern). Endpoint `type`/`title`/`plane` ride along so M7
renders a neighbor without a second fetch. New `NeighborEdge` / `NeighborCursor` / `NeighborStore`.

**Service — `GraphService`** (`server/app/graph/service.py`, alongside the existing
`DerivedEdgeGraph` write side):
- `neighbors(id, rel?, direction=both, cursor?, limit?)` → `NeighborPage{center_id, neighbors,
  next_cursor, rel, direction}`. Over-fetches `limit+1` to set `next_cursor` (an **opaque base64 of
  the compact-JSON keyset**) without a second round-trip; `limit` clamped to
  `[1, graph_neighbors_page_max]`. `InvalidDirection` (not out/in/both) and `InvalidCursor`
  (tampered/truncated/wrong-shape — the codec catches decode errors and rejects non-4-string
  arrays, never crashes) for task-4/router mapping. Unknown/isolated node → empty page (existence
  is the caller's concern).
- `build_context(id, depth?)` → `NodeContext{node, neighbors, depth, truncated, identity_capsule}`
  or `None` if unknown. `get_node` reused via a narrow **`NodeReader`** seam over `SearchService`
  (mirrors the chat `Retriever` pattern, rule 10) — so the center's body + profile + metadata come
  from the tested read path. Neighbor tree built from the capped `neighbors` primitive: `depth`
  clamped to `[0, build_context_max_depth]` (default max 2 = the 03-api `depth ≤ 2` contract), each
  visited node contributes at most `build_context_fanout` neighbors with a `truncated` flag (rest
  reachable via `traverse`), a **path-based `seen` cycle guard** so an A↔B two-cycle terminates
  (a repeat is listed but not re-expanded), depth-0 = node + capsule only (never touches the store).
  **L0 identity capsule is `None` here** — a clean seam; M5 task 2 distils it and wires it in.

**Config** (rule 9, no hardcoded caps): `graph_neighbors_page_default=25` / `_max=100`,
`build_context_default_depth=1` / `_max_depth=2` / `_fanout=10`. **Wired** into `app.state`
(`GraphService` over `PgNeighborStore` + the search service) + a `get_graph_service` dependency for
task 4 / M7.

**Verification.** Fakes replicating the store's ordering/keyset/filters — **16 new unit tests**
(`tests/test_graph_read_service.py`): ordering, both/in/out direction forwarding, rel-empty-is-no-
filter, limit clamping (default/max/floor incl. the `+1` over-fetch), a **cursor pagination
round-trip** + keyset encode/decode + malformed-cursor reject, unknown-node empty page; and
build_context depth-0/1/2, depth clamping to max, fanout truncation flag, and the cycle guard.
Plus **real-PG edge-SQL smoke** (`scripts/smoke_db.py`, the un-fakeable union/keyset/tombstone SQL):
`neighbors(ALEX)` ordered both-dirs with derived/tombstone endpoints excluded, endpoint type/plane
carried, rel + in/out filters, a real `::uuid` keyset page1→page2 with no overlap, and tombstone
exclusion on **both** ends (in-leg tombstoned src `neighbors(MEM2)==[]`; out-leg tombstoned dst
`neighbors(MEM1)==[ALEX]`). **The smoke needs the local pgvector Postgres running** — it's a
pre-push gate; not executed in this session (no local DB up), but it parses + lints clean and the
logic is covered by the fakes.

**449→499 tests green** (+16 unit; the rest of the delta is prior milestones), **ruff clean**,
app imports clean (no circular import from the `graph → search` service dependency).

**Independent review — APPROVE, no must-fix** (fresh agent, re-derived scope from 08 §M5 + 03-api +

## Task 2

**Identity capsule distiller + `build_context` L0 + M4 chat wiring**
([ADR-046](../adr/046-m5-mcp-server-oauth-connectors.md) §5 / [ADR-033](../adr/033-external-inspirations-obsidian-second-brain.md)
#1) — service layer + wiring, **local, not pushed**. Commits `2ea5834` (feature) + `cb3c25a`
(review minor).

The derived ~300-token "who the user is / current state" capsule — the compounding payoff of M5
task 1's `build_context` seam. A new **`app/identity/`** package, thin over the DB (no store git):

**Source reads — `PgCapsuleSourceStore`** (`server/app/identity/store.py`, plain asyncpg). The
**broadened source** (ADR-046 §5): `top_profile_hubs` ranks nodes that carry a derived profile by
**canonical graph degree** (a both-direction edge count on the node, tie-broken by profile freshness);
`recent_memories`/`recent_insights` return the newest live nodes of each kind (event-time
`occurred_start`, ingest-time fallback) with a first-chunk excerpt. **All three exclude tombstones**
(`merged_into IS NULL`). Insights are usually absent at M5 (they enrich as M6/M10 land) — an all-empty
source is a clean skip.

**Blob store — `PgIdentityCapsuleStore`** (same file). The derived blob `{text, generated_at,
source_refs}` in **`app_settings`** under one `identity_capsule` key (02-data-model §3) — **no
migration, rebuildable, rule-1 clean**. `current()` is the cheap read `build_context`/chat serve;
`save()` upserts (`ON CONFLICT`). `_decode_blob` tolerates absent/junk/empty-text → `None` (never
surfaces a broken capsule), folds a naive stored `generated_at` to UTC.

**Distiller — `IdentityCapsuleService`** (`server/app/identity/service.py`). One `conspect` call
(ADR-025 routing) turns the **fenced** (`<<<`/`>>>` data-not-instructions — injection hygiene, the
source is model-generated over captured content) blended source into the capsule; the authored system
prompt carries the ~300-token budget, the stored text is char-capped as a runaway backstop. **Best-
effort throughout (rule 7):** no source material / LLM down (`ProviderUnavailable`) / empty
distillation each return a *skipped* (not failed) outcome that **keeps the last capsule** — a stale
capsule beats none. **Single-flight** shared by two entry points (mirrors `ReindexService`):
`trigger()` (on-demand, background, `202 {run_id}`) + `run_scheduled()` (nightly + CLI, inline, never
raises); the slot is always released, the background task strong-ref'd + drained on shutdown. Each run
is wrapped in its `agent_runs` row (SUCCEEDED/FAILED with context — rule 7 visibility).

**Serving.** (1) `build_context` **L0**: `GraphService` gained a narrow `IdentityCapsuleReader` seam
(mirrors the task-1 `NodeReader` pattern) — the capsule is a cheap `app_settings` read set on every
`NodeContext` (depth 0 and deeper), **never distilled inline** (ADR-046 §5), best-effort so a read
failure omits L0 rather than failing the bundle. (2) **M4 chat**: `ChatService._answer` prepends the
capsule (via `chat/prompts.render_identity`, fenced) to the system prompt ahead of the CONTEXT block —
in-app chat finally consumes it; best-effort (a read failure answers without it). The `identity://me`
MCP resource stays **task 4** (reads the same blob).

**Scheduling / trigger.** `identity-capsule-refresh` joins the APScheduler roster at **04:35** (after
profile-refresh 04:10 + backfill 04:20 so it distils over the day's fresh hubs, before the 04:55
sweep — 04-pipelines) + `python -m app.cli identity-capsule-refresh` + **`POST
/admin/identity-capsule/refresh`** (`202 {run_id}`, `409` single-flight — the on-demand admin trigger,
following the existing bespoke `/admin/*` convention; the generic `POST /agents/{name}/run` surface is
unbuilt, an M8 ops-console concern). Config knobs (rule 9): source caps, token budget, char cap, cron.

**Verification.** **+25 unit tests** across four files (fakes only, no live LLM/DB — 08 policy):
distiller happy-path (source fenced + carried, blob saved with injected clock + provenance refs, run
SUCCEEDED), the three best-effort skips (no source / LLM down / empty → last capsule kept, no save),
config caps honored, char-cap truncation, single-flight (`trigger` background + `run_scheduled` skip),
blob encode/decode round-trip + junk/empty/naive-datetime rejection; `build_context` L0
present/absent/**failing-read survives**; chat capsule injection present/absent/**failing-read
answers**; scheduler job-spec; admin `202`/`409`. Plus **5 real-PG smoke checks** (`scripts/smoke_db.py`,
the un-fakeable SQL): degree-ranked hubs (ALEX degree 2, tombstoned GHOST excluded), recent-memory
ordering + excerpt, insight exclusion, blob save/current round-trip + `ON CONFLICT` upsert (dev-data-
safe backup/restore of any real `identity_capsule` row). **524 unit tests green**, **smoke 63/63**,
ruff clean, app imports/builds clean.

**Independent review — APPROVE-WITH-MINORS, no must-fix** (fresh agent, re-derived scope from 08 §M5
+ ADR-046 §5 / 033 / 02-data-model + traced the SQL, rule-7 paths, single-flight, and the boundary).
Confirmed: broadened source + blob shape + rule-1 rebuildability, all four best-effort paths keep the
prior capsule + close the run, single-flight matches reindex, both L0/chat reads cheap + best-effort +

## Task 3

**OAuth 2.1 authorization server + migration** ([ADR-046](../adr/046-m5-mcp-server-oauth-connectors.md)
§2) — server code only, **local, not pushed**. Commit(s) pending.

The self-hosted authorization server that gates the whole MCP surface. The `api` app becomes both
authorization server and (task 4) resource server; a connector self-registers, completes an
in-app password + consent flow, and gets opaque tokens — **no manual token distribution** (resolves
the old README cold-start #5 open question). New **`app/oauth/`** package + a root-mounted router.

**authlib scope.** ADR-046 says authlib covers "only the fiddly protocol crypto." Given the
async/asyncpg architecture (rule 5/8), wiring authlib's sync ORM-oriented `AuthorizationServer`
would fight the codebase; instead we use authlib's **primitives** — `create_s256_code_challenge`
(PKCE), `generate_token` (secure random codes/tokens/secrets), and the RFC 8414
`AuthorizationServerMetadata` schema (validates the discovery doc) — and hand-roll the flow
orchestration over our own store, exactly mirroring how web session tokens are already handled by
hand (`security.hash_session_token`). Nothing security-critical is hand-rolled; the audited crypto
is authlib's.

**Migration 010** (plain SQL, ADR-011). Three tables, all rebuildable operational state (never the
graph store): `mcp_oauth_clients` (DCR clients — `client_id` pk, optional `client_secret_hash`,
`metadata` jsonb), `mcp_auth_codes` (single-use PKCE-bound codes — **`code_hash` pk** [the code is a
credential → only its HMAC is stored], `code_challenge`, `consumed_at`, short `expires_at`), and
`mcp_tokens` (opaque access/refresh — `token_hash` unique, `kind`, `resource`, `expires_at`,
`revoked_at`, client FK-cascade + a client index). Up/down round-trip **verified against the live
dev pgvector** (tables + columns + index created, downgrade drops all three, re-upgrade clean).

**Store (`store.py`, `PgOAuthStore`).** Plain asyncpg. The load-bearing bit is **`consume_code`** —
an atomic `UPDATE … SET consumed_at = now() WHERE code_hash=$1 AND consumed_at IS NULL AND
expires_at > now() RETURNING …`, so a concurrent double-exchange loses the race and gets `None`
(single-use holds under concurrency, not a check-then-act). `consumed_code_client` distinguishes a
**replayed spent** code (→ revoke that client's tokens) from a never-issued one; `revoke_client_tokens`
/`revoke_all` return affected-row counts (the switch's `{revoked}`).

**Service (`service.py`, `OAuthService`).** All flow + crypto, unit-tested against the store
protocol:
- **DCR** — validates `redirect_uris` (non-empty; rejects `javascript:`/`data:`/`file:` schemes,
  allows https/http/native custom schemes for app callbacks), issues a `mcp_`-prefixed id; public
  (PKCE) clients get no secret, a confidential `token_endpoint_auth_method` mints one. Inert alone.
- **`load_authorization_request`** — two error shapes: an **`AuthorizeError`** (unknown client / bad
  or unregistered `redirect_uri`) must render a page and **never redirect** (open-redirect guard);
  once `redirect_uri` is exact-matched, everything else (`response_type`, missing/`plain` PKCE,
  unknown scope, wrong RFC-8707 resource) is an **`AuthorizeRedirectError`** carried back as a
  redirect with `error`/`state`. PKCE **S256 required** (`plain` rejected).
- **`exchange_code`** — consume (single-use) → client+redirect match → client auth (public no-op /
  confidential constant-time secret check) → **PKCE verify** (`create_s256_code_challenge` +
  `hmac.compare_digest`, verifier length-guarded) → issue an access (~1h) + refresh (60d) pair.
- **`refresh`** — validates, **rotates** (revokes the presented refresh, issues a fresh pair —
  sliding lifetime), and does **reuse detection**: a revoked/rotated-out refresh replay revokes the
  whole client.
- **`validate_access_token`** — the resource-server seam task 4's `/mcp` bearer gate reuses; fails
  closed (never raises) on missing/non-access/revoked/expired.
- **`revoke_all`** — the M5 switch.

**Router (`routers/oauth.py`, root-mounted — not under `/api/v1`).** Thin over the service (rule 5).
`GET /.well-known/oauth-authorization-server` + `/.well-known/oauth-protected-resource` (public
discovery), `POST /register` (open DCR, `201` RFC-7591 body), `GET /authorize` (renders the
server-side consent page — Approve/Deny, a password field unless a valid PWA session cookie
short-circuits it, all params carried as escaped hidden fields, a **double-submit CSRF cookie**),
`POST /authorize` (CSRF check → re-validate → deny-redirects / approve: **login-rate-limited**
password-or-session → issue code → redirect), `POST /token` (code-exchange + refresh, OAuth error
JSON with `Cache-Control: no-store`). Consent/error HTML is a pure `consent.py` (every dynamic value
`html.escape`d — client name/redirect are untrusted DCR input). Wired in `main.py`
(`app.state.oauth_service` over `PgOAuthStore`, `include_router(oauth.router)` **no prefix**) + a
session-gated **`POST /admin/mcp/revoke-all`** on the admin router.

**Config** (rule 9): `public_base_url` (discovery issuer + the RFC-8707 MCP resource id),
`mcp_token_hmac_secret` (the token/code HMAC — the MCP analogue of `session_secret`),
`mcp_oauth_scope` (`brain`), access/refresh/code TTLs. The `/authorize` password reuses `AuthService`
(new `verify_password` — verify without minting a web session) + the existing `login_rate_limiter`.

**Verification.** **+49 unit tests** (fakes only, 08 policy): `test_oauth_service.py` (31 — DCR
validation, all authorize error branches, the PKCE code round-trip + wrong/missing/short verifier,
single-use + replay-revoke, expiry, client/redirect mismatch, confidential secret, refresh rotation
+ reuse detection + expiry, token validation, revoke-all) and `test_oauth_router.py` (18 — discovery
shape, DCR `201`/`400`, consent render + password-hidden-on-session, CSRF mismatch, deny/approve
redirects, wrong-password `401` reprompt, `/token` exchange + refresh + `unsupported_grant_type`).
Plus **11 real-PG smoke checks** (`scripts/smoke_db.py`: the un-fakeable atomic single-use consume,
expired-code miss, `revoke_all` count + idempotency, client→codes/tokens FK cascade) — **74/74
green**. And a **scripted HTTP end-to-end** (throwaway harness: real ASGI app + real `PgOAuthStore`
against the live dev pgvector — discovery → DCR → `/authorize` PKCE+password consent → `/token` →
`validate_access_token` → refresh rotation → `/admin/mcp/revoke-all` → token now invalid, all PASS).
**573 unit tests green** (+49), ruff clean, app imports/builds clean, migration 010 up/down verified.
`authlib>=1.3` added to `pyproject` + `uv.lock`.

**Deferred to task 4/5 (by the ADR-046 task split):** the MCP server + `identity://me` + wiring
`validate_access_token` into a `/mcp` bearer gate (task 4); `MCP_TOKEN_HMAC_SECRET`/`PUBLIC_BASE_URL`
provisioning + Caddy root routes + Cloudflare passthrough (task 5). Until task 5 exposes the root
routes through Caddy, the endpoints are inert externally (Caddy proxies only `/api` + web today) — so
task 3 is deploy-safe on its own; migration 010 is additive. The **OAuth-focused independent security
review** is *also* re-run at the task-6 Accept gate over the whole surface.

**Independent review — APPROVE-WITH-MINORS** (fresh agent, re-derived scope from ADR-046 §2 + 03-api +
02-data-model + CLAUDE rules, adversarial security pass). Confirmed sound: PKCE S256 enforced + `plain`
rejected + constant-time verify, atomic single-use codes bound to client+redirect, no open-redirect on
any error path, opaque HMAC-hashed codes/tokens, double-submit CSRF + Lax cookies defeating the silent-
drive threat, DCR inert + scheme-guarded + escaped, fail-closed token validation, correct RFC error
codes, layering/config/migration invariants. **3 findings fixed** (commit — the review's must-consider
first two defeat/loosen invariants ADR-046 §2 names):
- **(1) refresh rotation was check-then-act** → a concurrent double-presentation of one refresh could
  mint two pairs with reuse-detection never firing. Made the **revoke the race-decider**:
  `revoke_token` now returns its rowcount; `refresh` mints the next pair only if the revoke flipped a
  row (1), else treats it as reuse → `revoke_client_tokens` + `invalid_grant`. Race-proof.
- **(2) revoke-all left outstanding auth codes live** (a code issued but not yet exchanged could still
  redeem after the switch) → `revoke_all` now also `invalidate_all_codes` (consume every live code)
  before flagging tokens, so the switch is *total*.
- **(3) DCR accepted `client_secret_basic`** but `/token` only reads the secret from the form body →
  such a client could never authenticate. Dropped it; only the two advertised methods
  (`none`/`client_secret_post`) register.
- **(4, logged — pre-existing, deploy-dependent)** the per-IP rate-limit key trusts the first
  `X-Forwarded-For` hop (spoofable). Shared with the existing web-login limiter (`routers/auth.py`);
  the correct trusted hop (e.g. `CF-Connecting-IP`) is a **task-5 / deploy decision** — fixing only the
  OAuth path would diverge from login. Deferred to task 5 as a project-wide auth-hardening item.
- **(5, logged — pre-existing, cosmetic)** Argon2 verify runs sync on the loop (rule-8 nit); identical
  to the existing login path, single-user load → left consistent with the codebase.
+4 unit tests (revoke-all-invalidates-pending-code, atomic-rotation reuse, basic-auth-rejected) + 3
smoke checks (`invalidate_all_codes`, `revoke_token` rowcount). **576 unit tests green** (+52 over task
2), **smoke 77/77**, E2E re-verified, ruff clean.

## Task 4

**MCP server + tools + rendering + capture source/burst**
([ADR-046](../adr/046-m5-mcp-server-oauth-connectors.md) §1/§3/§4 / [ADR-028](../adr/028-one-service-layer-mcp-peer-surface.md)) —
server code only, **local, not pushed**. Commit(s) pending.

The remote MCP server that exposes the service layer to external LLMs, gated by the task-3 OAuth
resource server. New **`app/mcp/`** package (four cohesive modules) + wiring in `main.py`.

**Transport + mount.** Official `mcp` SDK **`FastMCP`** with **Streamable HTTP** (SSE deprecated),
built in `create_app` (so the route is static) and mounted at the **root** via
`app.mount("/", mcp.streamable_http_app())` — the SDK registers a `Route` at exactly `/mcp` (not a
sub-`Mount`), so the spec path resolves with no trailing-slash redirect that would break connectors.
Mounted **last** so the `/api/v1`, OAuth, and discovery routes win; only `/mcp` +
`/.well-known/oauth-protected-resource/mcp` (the SDK serves this one) fall through. The transport's
`StreamableHTTPSessionManager` is entered in the app lifespan (`async with … .run()`) around the
existing startup/drain, so `/mcp` is live for the app's lifetime and torn down first on shutdown.

**Auth (resource server).** `mcp/tokens.py` `OAuthTokenVerifier` bridges the task-3
`OAuthService.validate_access_token` → the SDK's `TokenVerifier` protocol (`AccessToken`), taking a
**getter** for the service (built in `create_app`, service wired later in the lifespan). FastMCP's
`AuthSettings(issuer_url, resource_server_url=<base>/mcp, required_scopes=[brain])` drives its
`RequireAuthMiddleware`: an unauthenticated `/mcp` gets **401 + `WWW-Authenticate`** pointing at the
RFC 9728 `/.well-known/oauth-protected-resource/mcp` metadata (served by the SDK). No new auth logic
— the verifier fails closed via the task-3 check.

**Tools (thin, markdown at the boundary).** Seven tools registered in `mcp/server.py`, each a thin
closure over the app-state services (read lazily per call) that renders **LLM-optimized Markdown**
(`mcp/render.py`, pure functions): `search` (temporal `since`/`until`/`as_of` parsed from ISO,
`ProviderUnavailable`→friendly text), `get_node` (merged-tombstone redirect), `traverse`
(`InvalidDirection`/`InvalidCursor` mapped to guidance, cursor pointer), `build_context` (identity
capsule **L0** + node + capped neighbor tree), `list_planes`, `list_types`, and `capture`. **IDs are
rendered verbatim + labeled** (`id: <uuid>`), edge lists are **capped inline** at
`mcp_inline_edge_cap` (default 20) with a `traverse` overflow pointer (hub guard), node bodies are
already markdown → near-free. Read tools carry `readOnlyHint`; `capture` carries the write marker.

**Usage guidance.** The `initialize` **`instructions`** capsule (`mcp/text.py`, ~300 tokens: what
the brain is · the six tools + when to use each · the `search`→`build_context`→`capture` loop ·

## Task 6 (in progress) — live Accept

**Connector attempt 1 → OAuth OK but connect failed → diagnosed + fixed (2026-07-15).** User added the
custom connector in Claude pointing at `braindan.cc/mcp`, approved OAuth — Claude reported *"Your account
was authorized, but Braindan returned an error when connecting."*

**Diagnosis (no redeploy — captured live).** Added an on-demand **`diag.yml`** workflow (workflow_dispatch,
SSHes the VPS, dumps Bearer-redacted `docker compose logs api`; committed with `[skip ci]` so the api
container — holding the failure logs — was never recreated; the Dockerfile copies only `server/`, so a
`.github/` add can't bust the image anyway). The logs were decisive: the full OAuth flow succeeded
(`POST /register 201` → `GET/POST /authorize` → **`POST /token 200`**), then the authenticated
`POST /mcp` returned **`421 Misdirected Request`** with `WARNING mcp.server.transport_security Invalid Host
header: braindan.cc`.

**Root cause.** FastMCP's SDK enables **DNS-rebinding protection** and, when no `transport_security` is
passed, auto-allows **only localhost** — so behind the Caddy/Cloudflare edge the real `Host: braindan.cc`
is rejected. Crucially this bites **only after auth**: `RequireAuthMiddleware` is outer, so an
*unauthenticated* `POST /mcp` stops at 401 (never reaching the Host check) — which is why the earlier curl
probes looked clean and the bug only surfaced with a real token.

**Fix (`1ed0ee0`, deployed, CI green).** `build_mcp_server` now passes an explicit
`TransportSecuritySettings(enable_dns_rebinding_protection=True, allowed_hosts=[netloc, f"{netloc}:*"],
allowed_origins=[origin, f"{origin}:*"])` derived from `public_base_url` (rule 9 — no hardcoded host;
protection stays ON as defense-in-depth). **+1 regression test** (`test_transport_security_allows_the_public_host`)
asserts the public host is allowed. 600 tests green, ruff clean. Deployed; prod health all-green.
