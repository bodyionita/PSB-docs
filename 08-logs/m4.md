Per-task build narratives for **M4** — what was built, the independent review, and the

## Task 1

**Model routing engine** ([ADR-025](../adr/025-ui-editable-model-routing-and-per-task-effort.md) +
[ADR-043](../adr/043-quick-routing-tier-m4.md)) — server, local, **not pushed**.

**New `ModelRoutingService`** (`server/app/services/model_routing.py`) — the routing brain over
**three groups** (`chat`/`conspect`/`quick`), modeled identically per ADR-043 §1. Each group seeds
from config (`chat_chain`/`distill_chain`/`quick_chain` + `claude_max_effort`/`quick_effort`) and is
overlaid by user-saved routing in `app_settings` under one `model_routing` jsonb key
(`PgModelRoutingStore`, plain SQL over asyncpg — rule 5/ADR-011). In-memory cache **busted on save**
(`invalidate()`), so edits apply next request with no restart (ADR-025 §3). `resolve(group)` returns
a `RoutingDecision{chain, effort_by_provider}`: chain = `[active, fallback]` filtered to registered
chat providers, **degrading to the config seed chain (+ its effort) when a saved id is unknown/stale**
(rule 7); effort filtered to effort-capable providers only. `complete(group, messages,
requested_model=?)` delegates to the registry. `build_model_routing(settings, db, registry)` factory
shared by `main.py` + the CLI factories.

**Per-call effort through the provider boundary** (ADR-025 §4): `ChatProvider.complete` gains an
`effort` arg + a `supports_effort` class flag (`ClaudeMaxProvider` True → `effort or self._effort`;
`OpenAICompatibleProvider`/Nebius accepts-and-ignores, flag stays False). The registry threads
per-provider effort in `_chat_over_chain` and exposes a new `run_chain(messages, *, chain,
effort_by_provider, requested_model)` (the mechanics the service drives) plus `supports_chat` /
`supports_effort` accessors (so the service + a future GET /settings never hardcode which models
take effort). `registry.chat`/`distill` now delegate to `run_chain` — **behavior-preserved, the 21
registry-fallback tests stay green** (+1 new: `build_registry` registers `claude-max-sonnet`).

**`claude-max-sonnet` provider instance** (ADR-043 §3): a second `ClaudeMaxProvider` over the SAME
`claude` CLI with a config-driven `claude_max_sonnet_model` (default `claude-sonnet-4-6`; the exact
CLI alias is a one-line config swap confirmed at deploy), effort-capable, constructor effort seeded
to the cheap-tier `quick_effort`. `quick` seed = `[claude-max-sonnet, nebius]` @ low.

**6 conspect call sites rewired** through `routing.complete("conspect", …)`: capture organize +
nudge (`services/capture_pipeline.py`), entity resolution (`entities/resolver.py`), profile
generation (`entities/profile_refresh.py`), tag consolidation (`tags/service.py`), edge
consolidation (`vocab/edge_consolidation.py`). Services that still need the registry for a
non-routed leg keep it (profile-refresh embed, capture-pipeline transcribe); the three distill-only
services dropped it. Wired in `main.py` (`app.state.model_routing` + `get_model_routing` dependency)
and the standalone CLI factories (`services/reprocess.py`, `entities/profile_refresh.py`).

**No migration** — routing lives in the existing `app_settings` jsonb (ADR-043 §Consequences).

**Verification:** `ruff check` clean; **396 tests green** (+25: 13 new `test_model_routing.py` seed/
override/effort/degradation/cache-bust cases against fakes, +1 registry wiring test, +11 service
tests migrated to the routing seam). New unit tests use a `FakeModelRoutingStore` + `fake_routing`
helper (no live DB/LLM — 08 policy).

**Independent review — APPROVE, no must-fix** (fresh agent vs ADR-025/043 + CLAUDE.md rules + diff):
verified 3-group symmetry, effort routed to Claude tiers only, all 6 sites rewired with no residual
`registry.distill()` in app code, rule-7 degrade, no circular imports, CLI factories updated. Three
minors, **all fixed**: (1) sonnet constructor effort → `quick_effort` (was `claude_max_effort`);
(2) rule-7 degrade now also falls back to the seed's effort (effort read from whichever routing
supplied the chain); (3) added the `build_registry` wiring test.

Accept). Code committed locally, **not pushed** (push is the user's call — [09](../09-session-protocol.md)).

## Task 2

**Retrieval — hybrid vector+FTS RRF + recency prior + temporal filters** ([ADR-032](../adr/032-prior-art-adoptions.md)
§5/§7 + [ADR-037](../adr/037-profile-embedding-in-search-m3.md); 03-api §Search) — server, local, **not pushed**.

**Migration 008** (`008_fts_tsvector.py`): a generated **`tsvector`** column on **both**
`chunks` (`to_tsvector('english', content)`) and `node_profiles` (`… profile`), each `GENERATED
ALWAYS AS … STORED` + a **GIN** index (`chunks_tsv_gin` / `node_profiles_tsv_gin`). The FTS leg
mirrors the ADR-037 vector union so RRF fuses the **same** `chunks ⊍ node_profiles` node universe.
Generated + store-derived ⇒ no writer touches `tsv`; `POST /admin/reindex` and reprocess rebuild it
for free (Rule 1). `'english'` matches the asserted English corpus, so a non-English `tsquery`
self-suppresses (no matching lexemes) with **no language-detect dependency**. `down_revision=007`,
idempotent (`IF NOT EXISTS`), reversible downgrade; runtime head-check test moved 007→**008**.

**Hybrid `PgSearchStore.search_chunks`** (rewritten): two legs, each a best-per-node union over
`chunks ⊍ node_profiles`, bounded to a **candidate pool** (`LIMIT candidates`) before fusion —
- **VEC**: cosine `<=>` over `chunks.embedding ⊍ node_profiles.embedding`, `DISTINCT ON (node_id)`
  nearest row per node;
- **FTS**: `tsv @@ websearch_to_tsquery('english', query)` over `chunks.tsv ⊍ node_profiles.tsv`,
  best `ts_rank` per node.
Each leg is `row_number()`-ranked (deterministic `, node_id` tiebreak) and **`FULL OUTER JOIN`**ed
on `node_id`; **RRF** fuses by **rank, never raw score** (`1/(k+vec_rank) + 1/(k+fts_rank)`, `k=60`
— cosine and `ts_rank` are incommensurate, ADR-032 §5). A **mild recency prior** then multiplies the
fused score (bounded `[floor,1]`, `floor + (1-floor)·0.5^(age/half_life)` on `occurred_start ??
node_created_at`, future dates clamped to age 0, UTC-pinned `today`) **pre-cut** (ADR-032 §7). The FTS
leg self-suppresses on non-English / zero-lexeme queries (0 matching rows → vector-only ranking).
`websearch_to_tsquery` (not `to_tsquery`) never raises on arbitrary user text; params bound (no
injection). `min_score` now floors the **fused RRF×recency** score (default `0.0` ⇒ backward-safe;
scale note logged for the chat task below).

**Temporal filters** (03-api §Search, ADR-032): `since`/`until` = occurred-range window
(`COALESCE(occurred_end, occurred_start) >= since` / `occurred_start <= until`); `as_of` = simple
node-date `occurred_start <= as_of`. Applied to **all four sub-legs**; undated nodes fall outside any
window (literal-contract semantics). `SearchRequest` gains `since`/`until`/`as_of` (`date | None`),
threaded router → service → `RetrievalParams` (a value object bundling the RRF/recency knobs +
filters, so the store signature stays legible). Result shape (`SearchResultItem`) **unchanged**.

**Config** (Rule 9): `search_rrf_k` (60), `search_rrf_candidates` (60, clamped up to `top_k` so a
large `top_k` can't out-run the pool), `search_recency_half_life_days` (180, `Field(gt=0)` — SQL
divisor), `search_recency_floor` (0.9, `Field(ge=0, le=1)`).

**Verification:** `ruff` clean; **400 tests green** (+4 net: raw-query-unprefixed-to-FTS, candidate-
pool-≥-top_k both branches, temporal forwarding service+router; head-check 007→008). Hybrid SQL is
not fake-testable, so `scripts/smoke_db.py` gained **12 real-PG16 checks** (all green): FTS lexeme
match (`office`→{MEM1,MEM2}), non-English self-suppression → vector-only, and all three temporal
filters against seeded `occurred` dates. (Two **pre-existing, unrelated** smoke failures remain —
`list_entities`/`entities_touched_since` return real dev person nodes (Horia/Stefan) because the
local DB is now populated from the M3 prod reprocess; both checks predate this task and touch no
search code.)

**Independent review — APPROVE-WITH-MINORS, no must-fix** (fresh agent vs 08 §M4 kickoff-grill +
ADR-032/037 + 02/03/04 + CLAUDE rules + diff): confirmed rank-based RRF (no cosine/ts_rank blend),
per-leg candidate bounding, migration-008 spec + Rule-1 rebuildability, recency bounds, temporal
wording match on both legs, `websearch_to_tsquery` injection/crash safety, backward-safe `min_score`,
no ORM leak / config-only / result-shape-unchanged. **4 minors, 3 fixed:** div-by-zero guard on the
half-life (`Field(gt=0)` + floor bounds), UTC-pinned recency `today` (was `CURRENT_DATE`),
deterministic rank tiebreak (`, node_id`). **1 logged as a follow-up (MINOR-1):** the chat task's
"not in your memories" `min_score` floor (04-pipelines §5) must be tuned to the fused RRF×recency
scale (~0.03 max), **not** a cosine scale — a note for M4 task 3, not a defect here.
