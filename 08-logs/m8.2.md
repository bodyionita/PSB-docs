## Task 1 — Temporal engine (symbolic schema · resolver · tokens · render)

**The pure-logic core of "LLMs classify, code computes"** ([ADR-056](../adr/056-temporal-correctness-date-tokens.md),
CLAUDE.md hard rule 12). First task in the milestone; **no migration, no DB, no LLM, no wiring** —
all downstream consumers (organizer T2, indexer/chat/MCP/capsule T3, web mirror T4) build on this.
New package `server/app/temporal/`. **Committed locally (`d30d6b6` + review fixes `2787701`), not
pushed.**

### What was built

- **`symbolic.py` — the emission contract.** A fail-closed pydantic discriminated union (on `kind`)
  of exactly what the organizer LLM may emit — never a computed date, only a *classification*:
  `ExplicitRef` (whole/partial calendar date, year optional → code snaps), `RelativeRef`
  (`unit` day/week/month/year × signed `offset`), `WeekdayRef` (weekday × last/this/next),
  `MonthRef` (month × year|direction), `SeasonRef` (season × year|year_offset). `extra="forbid"`
  guards against silent drift; validators coerce common variants ("Tuesday"→`tue`, "fall"→`autumn`).
  `parse_symbolic(data)` returns `None` on anything invalid — **never raises** on bad model output.
- **`resolver.py` — the deterministic compute.** `resolve(ref, anchor)` / `resolve_reference(data,
  anchor)` turn a symbolic ref into an absolute `ResolvedTime` against the capture's **stored
  anchor** (never wall-clock — reprocess-determinism, ADR-056 §1): offset arithmetic, weekday walks
  (last/this/next, `or 7` so same-weekday goes back a full week), month/year snapping, yearless
  snap to the most-recent-past occurrence (year-by-year back-walk, 8-year reach covers a century
  leap gap), northern-hemisphere season windows (winter spans the year boundary). Anything it can't
  compute to a real date → `None` (fail-closed, rule 12). Pure stdlib (`datetime`/`calendar` +
  integer month math), **no `dateutil`** — so the T4 web mirror computes byte-identical results.
- **`tokens.py` — the `[[t:START[/END][|label]]]` token + `PartialDate`.** Partial-ISO at honest
  granularity (`2025` / `2025-07` / `2025-07-07` / `2025-07-07T22:00`); parse/serialize, `find_tokens`
  locates every token in a body (malformed → `resolved=None`, surfaced not crashed), and the
  **day-granular `occurred` floor/ceil** the DB stores (`occurred_*` stay `date` — tokens own sub-day,
  ADR-056 §6; a coarse single partial honestly spans its period for the ADR-049 dedup gate, a
  precise day/minute is a point → `occurred_end` None).
- **`render.py` — tokens → text, never shown raw** (ADR-056 §4). `render_relative` = the live phrase
  ("10 days ago", "last month", "summer 2025") computed at *now*, with `render_absolute` as the
  tooltip; `expand_for_index` = stable absolute only (vectors see language, not token noise);
  `expand_for_llm` = absolute + a fresh relative hint ("7 July 2026 (10 days ago)") for the LLM-bound
  rendering contract. Whole-body helpers replace every token. The humanization spec is documented for
  the web mirror; **rounding pinned to round-half-up** (`math.floor(x+0.5)`) so JS `Math.round` agrees
  at `.5` ties.

## Task 2 — Organizer v6 + migration 016 (anchor · symbolic time → occurred + tokens · interiority)

**Where the temporal engine (T1) and the interiority marker meet the organizer** —

classify, code computes", inline tokens, date-granular `occurred`) +
[ADR-055](../adr/055-interiority-inner-voice-first-class.md) §1/§2 (the marker + inner-voice

### Grill decision resolved this session (inner-voice edge)

ADR-055 §2 requires the extracted `internal` node to be **"edge-linked to the event node it arose
from,"** but the codebase had **no intra-capture content→content edge mechanism** (edges only ever
targeted resolved *entity hubs*) and the ADR **named no rel**. Surfaced to the user as an explicit
fork; **user chose: reuse the seeded `led_to` rel** — the edge sits on the **EVENT** node → the
internal node, and the organizer references the sibling by **local index** (`arose_from`); **no new
vocabulary** (aligns with §2's "existing types" minimalism — a later `arose_from` rel or a
`feeling`/`reflection` type stays a trivial migration since internal nodes are already identifiable).
Recorded here + in [../08-implementation-plan.md](../08-implementation-plan.md) §M8.2 Task 2 (no new
ADR — a mechanical elaboration within ADR-055 §2, not a new architectural axis).

### What was built

- **Organizer prompt v6** (`app/capture/organizer.py`). New `{anchor}` line
  (`render_anchor` — "recorded on Thursday, 2026-07-17 08:40 (Europe/Bucharest)", the capture's
  **stored** time, never wall-clock → reprocess-deterministic). The emitted JSON drops the old
  LLM-computed `occurred` and gains **`time_refs`** (a list of symbolic classifications — the exact
  five `symbolic.py` kinds with a `phrase` + an optional `event` flag), **`interiority`**
  (`internal|external|mixed`), and **`arose_from`** (the local index of the sibling event node an
  `internal` node came from). The prompt spells out that the model **only classifies** each time
  phrase (never a date), keeps the phrase as prose, and marks recurrence as prose (no token).
- **`validate_organizer_output` gained an `anchor` param + a two-pass build.** Pass 1 builds each
  surviving node and resolves its `time_refs` (`_resolve_time_refs` → the T1 `resolve_reference`):
  every resolvable ref's `phrase` is spliced out of the body for its `[[t:…]]` token (overlap-safe
  first-occurrence splice; unresolvable → **no token, phrase stays prose**, rule 12), and the `event`
  ref supplies the **day-granular** `occurred`/`occurred_end` (`_date_iso` drops any `T..:..` — tokens
  own sub-day, §6). `interiority` validates to the three values (`external` default; hubs left
  unstamped). Pass 2 **remaps `arose_from`** from the LLM's raw node index to the surviving node's
  result index (dangling/self → `None`), so a dropped earlier node can't misdirect the edge.
- **Pipeline** (`services/capture_pipeline.py`). `created_local` is computed **before** organize at
  all three call sites (`_process`, `reprocess_capture`, `_replace_notes_via_reorganize`) and threaded
  as the **single anchor** into both the prompt and validation. `_build_documents` stamps
  `interiority`/`occurred_end` on content nodes and draws the **event `led_to` internal** sibling edge
  from `arose_from` (bounds-checked, non-self; `_INNER_VOICE_REL = "led_to"`).
- **Persistence.** `NodeDocument.interiority` (None on hubs) renders a frontmatter `interiority:` line
  (`node_writer`, `ORGANIZER_VERSION` v3→**v4**); the frontmatter parser reads it back
  (`_clean_interiority` → NULL on unknown); `NodeUpsert.interiority` + the `nodes` upsert SQL (`$19`,
  ON CONFLICT) persist it; the indexer passes `meta.interiority`. **Migration 016** adds the nullable
  `nodes.interiority` column (additive, reversible, no backfill — reprocess fills it, Rule-1
  rebuildable).
