## Task 1 ‚Äî Temporal engine (symbolic schema ¬∑ resolver ¬∑ tokens ¬∑ render)

**The pure-logic core of "LLMs classify, code computes"** ([ADR-056](../adr/056-temporal-correctness-date-tokens.md),
CLAUDE.md hard rule 12). First task in the milestone; **no migration, no DB, no LLM, no wiring** ‚Äî
all downstream consumers (organizer T2, indexer/chat/MCP/capsule T3, web mirror T4) build on this.
New package `server/app/temporal/`. **Committed locally (`d30d6b6` + review fixes `2787701`), not
pushed.**

### What was built

- **`symbolic.py` ‚Äî the emission contract.** A fail-closed pydantic discriminated union (on `kind`)
  of exactly what the organizer LLM may emit ‚Äî never a computed date, only a *classification*:
  `ExplicitRef` (whole/partial calendar date, year optional ‚Üí code snaps), `RelativeRef`
  (`unit` day/week/month/year √ó signed `offset`), `WeekdayRef` (weekday √ó last/this/next),
  `MonthRef` (month √ó year|direction), `SeasonRef` (season √ó year|year_offset). `extra="forbid"`
  guards against silent drift; validators coerce common variants ("Tuesday"‚Üí`tue`, "fall"‚Üí`autumn`).
  `parse_symbolic(data)` returns `None` on anything invalid ‚Äî **never raises** on bad model output.
- **`resolver.py` ‚Äî the deterministic compute.** `resolve(ref, anchor)` / `resolve_reference(data,
  anchor)` turn a symbolic ref into an absolute `ResolvedTime` against the capture's **stored
  anchor** (never wall-clock ‚Äî reprocess-determinism, ADR-056 ¬ß1): offset arithmetic, weekday walks
  (last/this/next, `or 7` so same-weekday goes back a full week), month/year snapping, yearless
  snap to the most-recent-past occurrence (year-by-year back-walk, 8-year reach covers a century
  leap gap), northern-hemisphere season windows (winter spans the year boundary). Anything it can't
  compute to a real date ‚Üí `None` (fail-closed, rule 12). Pure stdlib (`datetime`/`calendar` +
  integer month math), **no `dateutil`** ‚Äî so the T4 web mirror computes byte-identical results.
- **`tokens.py` ‚Äî the `[[t:START[/END][|label]]]` token + `PartialDate`.** Partial-ISO at honest
  granularity (`2025` / `2025-07` / `2025-07-07` / `2025-07-07T22:00`); parse/serialize, `find_tokens`
  locates every token in a body (malformed ‚Üí `resolved=None`, surfaced not crashed), and the
  **day-granular `occurred` floor/ceil** the DB stores (`occurred_*` stay `date` ‚Äî tokens own sub-day,
  ADR-056 ¬ß6; a coarse single partial honestly spans its period for the ADR-049 dedup gate, a
  precise day/minute is a point ‚Üí `occurred_end` None).
- **`render.py` ‚Äî tokens ‚Üí text, never shown raw** (ADR-056 ¬ß4). `render_relative` = the live phrase
  ("10 days ago", "last month", "summer 2025") computed at *now*, with `render_absolute` as the
  tooltip; `expand_for_index` = stable absolute only (vectors see language, not token noise);
  `expand_for_llm` = absolute + a fresh relative hint ("7 July 2026 (10 days ago)") for the LLM-bound
  rendering contract. Whole-body helpers replace every token. The humanization spec is documented for
  the web mirror; **rounding pinned to round-half-up** (`math.floor(x+0.5)`) so JS `Math.round` agrees
  at `.5` ties.

### Grill decision resolved this session (inner-voice edge)

ADR-055 ¬ß2 requires the extracted `internal` node to be **"edge-linked to the event node it arose
from,"** but the codebase had **no intra-capture content‚Üícontent edge mechanism** (edges only ever
targeted resolved *entity hubs*) and the ADR **named no rel**. Surfaced to the user as an explicit
fork; **user chose: reuse the seeded `led_to` rel** ‚Äî the edge sits on the **EVENT** node ‚Üí the
internal node, and the organizer references the sibling by **local index** (`arose_from`); **no new
vocabulary** (aligns with ¬ß2's "existing types" minimalism ‚Äî a later `arose_from` rel or a
`feeling`/`reflection` type stays a trivial migration since internal nodes are already identifiable).
Recorded here + in [../08-implementation-plan.md](../08-implementation-plan.md) ¬ßM8.2 Task 2 (no new
ADR ‚Äî a mechanical elaboration within ADR-055 ¬ß2, not a new architectural axis).

### What was built

- **Organizer prompt v6** (`app/capture/organizer.py`). New `{anchor}` line
  (`render_anchor` ‚Äî "recorded on Thursday, 2026-07-17 08:40 (Europe/Bucharest)", the capture's
  **stored** time, never wall-clock ‚Üí reprocess-deterministic). The emitted JSON drops the old
  LLM-computed `occurred` and gains **`time_refs`** (a list of symbolic classifications ‚Äî the exact
  five `symbolic.py` kinds with a `phrase` + an optional `event` flag), **`interiority`**
  (`internal|external|mixed`), and **`arose_from`** (the local index of the sibling event node an
  `internal` node came from). The prompt spells out that the model **only classifies** each time
  phrase (never a date), keeps the phrase as prose, and marks recurrence as prose (no token).
- **`validate_organizer_output` gained an `anchor` param + a two-pass build.** Pass 1 builds each
  surviving node and resolves its `time_refs` (`_resolve_time_refs` ‚Üí the T1 `resolve_reference`):
  every resolvable ref's `phrase` is spliced out of the body for its `[[t:‚Ä¶]]` token (overlap-safe
  first-occurrence splice; unresolvable ‚Üí **no token, phrase stays prose**, rule 12), and the `event`
  ref supplies the **day-granular** `occurred`/`occurred_end` (`_date_iso` drops any `T..:..` ‚Äî tokens
  own sub-day, ¬ß6). `interiority` validates to the three values (`external` default; hubs left
  unstamped). Pass 2 **remaps `arose_from`** from the LLM's raw node index to the surviving node's
  result index (dangling/self ‚Üí `None`), so a dropped earlier node can't misdirect the edge.
- **Pipeline** (`services/capture_pipeline.py`). `created_local` is computed **before** organize at
  all three call sites (`_process`, `reprocess_capture`, `_replace_notes_via_reorganize`) and threaded
  as the **single anchor** into both the prompt and validation. `_build_documents` stamps
  `interiority`/`occurred_end` on content nodes and draws the **event `led_to` internal** sibling edge
  from `arose_from` (bounds-checked, non-self; `_INNER_VOICE_REL = "led_to"`).
- **Persistence.** `NodeDocument.interiority` (None on hubs) renders a frontmatter `interiority:` line
  (`node_writer`, `ORGANIZER_VERSION` v3‚Üí**v4**); the frontmatter parser reads it back
  (`_clean_interiority` ‚Üí NULL on unknown); `NodeUpsert.interiority` + the `nodes` upsert SQL (`$19`,
  ON CONFLICT) persist it; the indexer passes `meta.interiority`. **Migration 016** adds the nullable
  `nodes.interiority` column (additive, reversible, no backfill ‚Äî reprocess fills it, Rule-1
  rebuildable).

### E ¬∑ `occurred-enrichment` review kind (ADR-056 ¬ß7)

- **The nightly flagger** ‚Äî new `app/services/occurred_enrichment.py`: `OccurredEnrichmentService`
  (DB-only, like graph-health/maybe-digest ‚Äî its own `agent_runs` row, never raises, rule 7) files
  one `occurred-enrichment` review item per **undated content node** returned by
  `PgOccurredEnrichmentStore.undated_content_nodes` (live, non-`inbox/`, **content** type ‚Äî not an
  entity hub, `occurred_start IS NULL`, recent-first, **bounded** by `occurred_enrichment_max_per_run`
  = 20). **Idempotent (rule 6):** a `NOT EXISTS` excludes any node that already has an item in an
  **already-handled** status ‚Äî `pending`/`maybe`/**`discarded`** ‚Äî so a rerun never double-files and a
  *dismiss sticks* (a resolved node self-excludes via `occurred_start IS NULL`). Payload = `{node_id,
  title, type}` (a node **id**, not names ‚Äî a reprocess rebuilds nodes, so a stale item is harmless).
  **Scope note (recorded, not silent):** the first cut flags the unambiguous **undated** case
  (`occurred_start IS NULL` ‚Äî the "graph-health already counts them" signal); "coarsely-dated" is
  genuinely underspecified against the day-granular `occurred_*` columns (a real year-long event is
  indistinguishable from a year-granular guess), so refining it into a separate flag is a logged
  follow-up, not folded in.
- **The NL classifier** ‚Äî new `app/services/nl_time.py`: `NlTimeClassifier.classify(phrase, anchor)`
  runs one `conspect` call whose prompt asks for a **single symbolic time-ref** (the *exact* schema the
  organizer emits, minus `event`; rule 12 ‚Äî the model never emits a date) then the **T1 resolver**
  computes the absolute date against the anchor. **Fail-closed:** a down provider, a `{"kind":"none"}`,
  or an unresolvable classification ‚Üí `None`. Reuses the organizer's `render_anchor` + `parse_organizer_json`.
- **The resolution** ‚Äî `ReviewService` gains a `time_classifier` dep + an `answer` param;
  `_resolve_occurred_enrichment` runs the answer through the classifier **anchored to now (local)** ‚Äî
  a live user correction, not a replayed path, so wall-clock is legal (rule 12) ‚Äî then applies the
  **mechanical tier of ¬ß5** (`NodeWriter.set_occurred` day-granular partial-ISO + re-index + commit).
  `maybe` parks; `skip`/`unknown`/`dismiss` ‚Üí discarded (node stays undated); an uninterpretable
  answer ‚Üí `400` (item stays decidable, rephrase); a node reprocessed away since filing ‚Üí discarded
  (stale). The `answer` threads through the router + `resolve_batch` (a batch `skip` clears many).
