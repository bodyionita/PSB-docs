## Task 1

**Server — `GET /nodes/{id}/neighbors`** (the grouped map endpoint,
[ADR-051](../adr/051-m7-map-build-decisions.md) §2 as amended by
[ADR-052](../adr/052-map-zones-keyed-by-rel.md)). One route, two modes, thin over the M5
`GraphService.neighbors` primitive — no new store surface, no migration. **Committed locally, not
pushed.**

### What was built

- **`PgNeighborStore.center_header`** — a lightweight `{node_id, type, title, plane, planes}` read
  for the `center` echo, so a re-center doesn't pay `get_node`'s store-file read.
- **`PgNeighborStore.neighbor_zones`** — the grouped window query: the same both-directions union as
  the M5 `neighbors` primitive, but `ROW_NUMBER() OVER (PARTITION BY rel ORDER BY origin, dir,
  node_id)` caps each zone to the first `map_zone_fanout` neighbors and `COUNT(*) OVER (PARTITION BY
  rel)` carries the zone's true size. Direction filter sits before the window (so ranking + count are
  direction-scoped); tombstoned endpoints excluded both legs; `until`-closed edges returned. One
  round-trip returns every zone already capped — bounded regardless of a hub's degree.
- **`GraphService.neighbor_zones`** — groups the rows by `rel` (`itertools.groupby` over the
  `(rel, origin, dir, node_id)`-ordered rows), building a per-zone `total` + `next_cursor` (the M5
  keyset of the zone's last returned neighbor, `None` when the zone fit). The `?rel=` "show more"
  mode reuses `GraphService.neighbors` **untouched** — a zone *is* a rel, so the rel-only cursor
  resumes it exactly.
- **Router** `GET /nodes/{node_id}/neighbors` (in the search & graph router): `uuid.UUID` path
  (422 on malformed), `direction` a `Literal` (auto-422), bad cursor → 422, unknown node →
  `center=null` + empty zones. `response_model=None` (the two-mode `NeighborZonesResponse |
  NeighborPageResponse` union would mis-serialize under Pydantic smart-union — the page shape would
  validate as an empty zones shape — so the route returns the model instance directly; **logged for
  task 2**: the web types this endpoint by hand from 03-api rather than the OpenAPI schema).
- Config `map_zone_fanout` (seed 8, rule 9); Pydantic `MapNeighborItem` / `MapZone` /
  `NeighborCenter` / `NeighborZonesResponse` / `NeighborPageResponse`.

### Mid-implementation replan → ADR-052

The independent review caught a real correctness gap: ADR-051 §2 keyed a zone by `(origin, rel)`
but specified that "show more" pages by the **rel-only** cursor "untouched" — the two diverge for
the sole dual-origin rel, **`similar`** (a human `link` writes a *canonical* `similar`;
[ADR-049](../adr/049-dedup-sweep-merge-core-build-decisions.md); the nightly recompute writes a
*derived* `similar`, and the two coexist by design). "Show more" on a canonical-`similar` zone would
bleed derived-`similar` rows in (rendered solid, duplicating the derived zone). Per
[09](../09-session-protocol.md) (hit an unrecorded/contradictory decision → replan, don't decide
inline) this became a short `/grilling` replanning pass → **[ADR-052](../adr/052-map-zones-keyed-by-rel.md)**:
**zones are keyed by `rel` alone**, so `similar` collapses to one zone; per-neighbor `origin` still
drives the solid/faint styling (§6); the zone-level `origin` field is dropped. The M5 primitive,
"show more" mode, and cursor codec stay untouched — the rel-only cursor is now correct by
construction. `similar` is the **only** dual-origin rel (confirmed from `compute_similar` /
`replace_derived_edges`, which only ever write `origin='derived', rel='similar'`), so every other
zone is unchanged.

- **737 unit tests + 129 real-PG smoke green, ruff check + format clean.**

## Task 2

**Web — the canvas map** ([ADR-051](../adr/051-m7-map-build-decisions.md) /
[ADR-052](../adr/052-map-zones-keyed-by-rel.md); 06 §3c). A `react-force-graph-2d` neighborhood
explorer over the task-1 `GET /nodes/{id}/neighbors`. **Client-only (ADR-006 / rule 4 — no server
code touched); committed locally, not pushed.** Commits `93ac37f` (feature) / `624a644` (review
fixes).

### What was built

- **`react-force-graph-2d@1.29.1`** added (the 2D-only build, no three.js — ADR-032 #12 "bundle-
  lean"). Installed via `corepack pnpm@9.15.0` (the repo's pinned pnpm; the machine's global pnpm 7
  hits an `ERR_INVALID_THIS` registry bug on Node 24).
- **`features/map/`** — the feature:
  - **`graphModel.ts`** — turns the center + its rel-keyed zones into force-graph nodes/links,
    assigning each zone an angular **sector** (target `tx/ty`) so a custom force settles "people one
    side, topics another"; a per-zone **"+N" node** when `total > shown` (carries the zone `rel` +
    `next_cursor`); link `source/target` set from each neighbor's `dir` (out = center→neighbor, in =
    neighbor→center) for correct arrowheads; per-neighbor `origin`/`until` carried for styling.
  - **`MapCanvas.tsx`** — the canvas: a pinned focal node (`fx/fy=0`), a **custom `zone` d3-force**
    pulling neighbors to their sector (stock charge/link kept for organic spacing), **emoji=type**
    marks (reuses `ui/nodeTypes`), **plane-coloured halo + hub ring** (hub vs content from the
    governed `entity_like_types`, not a hardcoded list), the **theme accent reserved for the focal
    node**; edges **canonical solid+arrowhead / derived faint no-arrow / superseded (`until`)
    dashed+dimmed** (arrowhead shares the per-edge colour); hover peek via native tooltips;
    zoom-gated titles; single-click → re-center / focal → drawer / "+N" → show-more.
  - **`MapScreen.tsx`** — breadcrumbs (crumb click truncates forward history), the focal **caption
    chip**, the **`NodePreview` drawer** on focal-click (rule 10 reuse; its edges re-center),
    per-zone **show-more** merged into local state (append without refetching the neighborhood; node
    objects reused across renders so positions don't reset), and the **empty state** (embedded
    search + restore-last-centered from `localStorage`).
  - **`useMap.ts`** (`useNeighbors`, `keepPreviousData` so a re-center swaps in place) · **`mapNav.ts`**
    (the `MapNavContext` — kept out of `ui/` so `NodePreview` stays feature-agnostic via an optional
    `onOpenNode`).
- **`ui/planeColors.ts`** — the new **theme-independent** categorical plane palette (ADR-051 §4): the
  6 seed planes get fixed hues, a governed/unknown plane hashes to a stable hue, inbox/none is
  neutral grey. Not derived from the theme accent (a plane means the same thing in every theme).
- **Shell** (`AppShell.tsx`) — the **8th "Map" tab** with a per-tab **`wide`** flag that breaks the
  map out of the 640px reading column to a full-viewport canvas; a lifted one-shot **`mapSeed`** +
  `MapNavContext` provider.
- **Entry points** (ADR-051 §8) — **"Explore in map"** on Search cards + **navigable `NodePreview`
  edge rows** in Search **and** Chat (the shared primitive gained an optional `onOpenNode`; it still
  imports no feature module).

## Task 3

**Web — list fallback + reduced-motion + phone** ([ADR-051](../adr/051-m7-map-build-decisions.md)
§7). The tappable-list renderer for the Map: the same rel-keyed zones the canvas draws, rendered as
calm grouped tappable rows — both the `prefers-reduced-motion` fallback and a manual Canvas/List
toggle. **Client-only (ADR-006 / rule 4 — no server code touched); committed locally, not pushed.**
Commits `78ec92f` (feature) / `469d504` (review fix).

### What was built

- **`features/map/MapList.tsx`** — the grouped list renderer over the shared `EffectiveZone[]`:
  - A **focal header** (tap → the shared `NodePreview` drawer, rule 10 reuse) standing in for the
    canvas caption chip: emoji=type + title + type label + plane badge.
  - **One `<section>` per rel zone** (heading = humanised rel + the zone `total`), each with tappable
    **neighbor rows** (tap → re-center) and a per-zone **"Show N more"** button (`remaining =
    total − shown`, calls the same `onShowMore(rel, next_cursor)` the canvas "+N" node uses).
  - **Inline edge styling mirroring the canvas** (ADR-051 §6): derived rows read faint + "similar"
    with **no direction arrow** (symmetric); superseded (`until`) rows read **dashed + dimmed** with
    the `until` date; canonical rows carry a **direction arrow** (→ out / ← in). Hub rows (governed
    `entity_like_types`) render heavier.
- **`MapScreen.tsx`** — the wiring:
  - **`useReducedMotion`-driven default** — `view = viewOverride ?? (reducedMotion ? 'list' :
    'canvas')`: under OS reduced-motion the list is the default (the force sim doesn't run — ADR-051
    §7 consequence); otherwise canvas. A manual **`ViewToggle`** (Canvas/List, in the header) sets a
    sticky override on top of that default.
  - **Extracted `effectiveZones`** (query zones ∪ locally-appended "show more" neighbors) as the one
    memo both the canvas (`buildGraph`) and the list read, so the two views stay in lockstep.
  - **`busyRels`** render-state mirrors the existing `showMoreBusy` ref (ref = synchronous
    double-click guard; state = the list button's "Loading…" label).

### Independent review

A fresh, high-effort agent reviewed the `78ec92f` diff against ADR-051/052 + 08 §M7 + 06 §3c +

stale docstring. **Minors logged:** a small `isHub` helper duplication between `MapList` and
`MapCanvas` (rule-10 nit, a shared `ui/nodeTypes` helper would remove it) — follow-up.
